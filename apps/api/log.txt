<h3>
  <a name="what-is-a-raw-github-endraw-repository" href="#what-is-a-raw-github-endraw-repository">
  </a>
  üìå What is a <code>.github</code> Repository?
</h3>

<p>A <strong>special</strong> repository that centralizes GitHub workflows, templates, and automation for an organization or user account.</p>


<hr>

<h3>
  <a name="why-use-it" href="#why-use-it">
  </a>
  üöÄ Why Use It?
</h3>

<p>‚úÖ <strong>Standardize Issues &amp; PRs</strong> ‚Üí Issue &amp; PR templates (<code>.github/ISSUE_TEMPLATE/</code>, <code>.github/PULL_REQUEST_TEMPLATE.md</code>)</p>

<p>‚úÖ <strong>Reuse GitHub Actions</strong> ‚Üí Store workflows (<code>.github/workflows/</code>) for all repos</p>

<p>‚úÖ <strong>Centralize Policies</strong> ‚Üí CODE_OF_CONDUCT.md, SECURITY.md, CONTRIBUTING.md</p>

<p>‚úÖ <strong>Funding &amp; Sponsorship</strong> ‚Üí <code>.github/FUNDING.yml</code> to show sponsor links</p>

<p>‚úÖ <strong>Repository Overview</strong> ‚Üí Use <code>.github/README.md</code> as a landing page for organization-wide documentation</p>


<hr>
